GitHub Pages

El espacio web gratuito para vos, tu organización y tus proyectos

GitHub Pages para usuarios u organizaciones

Creamos un repositorio con nuestro nombre de usuario o nombre de la organización seguido de .github.io

Ya con el repositorio creado tenemos 2 opciones para trabajar con nuestros commits

Clonar un repositorio ya existente

$ git clone https://github.com/nombreDeUsuario/nombreDeUsuario.github.io

Vincularlo a un repositorio local

$ git remote add origin https://github.com/nombreDeUsuario/nombreDeUsuario.github.io

GitHub Pages para proyectos

Creamos un repositorio nuevo o un branch en nuestro repositorio.

Creamos el nuevo branch con el nombre gh-pages

$ git branch gh-pages

El commit que queremos que sea visible lo guardaremos en nuestro nuevo branch.

$ git push origin gh-pages

Todos los commits y cambios que hagamos en la rama gh-pages, se verán reflejados en esta URL. GitHub Pages es de las mejores formas de mostrar nuestros proyectos al mundo a través de un sitio web gratuito. Podemos usarlo para un portafolio, landing page e incluso para crear un blog. Todo de manera gratuita y con repositorios ilimitados.

Para ver nuestros proyectos ingresamos en la ruta: http://nombreUsuario.github.io/nombreRepositorio

Más sobre los Branch

Los Branch son ramas dentro de nuestro árbol de trabajo del repositorio. Estos nos permiten tener diferentes opciones de un mismo proyecto o permitirnos trabajar en equipo con un brach diferente cada uno y después unirlos.

Comandos:

  • Ver nuestros Branch locales $ git branch
  • Crear Branch local $ git checkout -b nombreBranch
  • Seleccionar brach para trabajar $ git checkout nombreBranch
  • Eliminar branch $ git checkout -d nombreBranch

El branch recién se crea en nuestro repositorio remoto una vez que realizamos el push por primera vez: $ git push origin nombreBranch
